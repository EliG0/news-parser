# Краткое обьяснение работы программы (на лисах)

Представим, что вы лисичка, которая очень фанатеет от Лисартамонова, Лисаков, Лисаркиной и других деятелей, посему
очень хочет узнавать новости про них - первой. Для этого у нее есть специальная программа,
которая умеет искать и собирать информацию с разных сайтов по ключевым словам. Лисичка устанавливает и запускает
программу, и попадает на главный экран. Но интерфейс пока не предусматривает адекватный UI/UX, посему лисичке приходится
зайти в админ-панель. В ней, она заходит в раздел источников, и добавляет туда различные источники, с которых она
хочет собирать новости. Так у лисички получился список источников с самыми разными типами, от RSS-ленты
(например с портала Лисецк48.ру) до обычных сайтов, к которым нужно применять регулярные
выражения (например ЛисецкаяОбласть.рф). После того как лисичка добавила все источники, ей придется добавить ключевые
слова (например, "Лисартамонов", "Лисин", "Лисов", "Лисицин", "лисил", "лиснул", "лисанул" - не важно). После добавления
источников и ключевых слов она может наконец зайти на главный экран и нажать на эту заветную кнопку:

1. При нажатии на кнопку, в файле `tasks.py` происходит вызов функции
   `celery_crawl_all_active_sources_task`: программа получает список всех активных источников и запускает обработку
   каждого из них (`celery_crawl_single_source(source_id)`), то есть - лезет в БД, достает эту
   статью по `source_id`, ищет в кэше кейворды (`get_cached_keywords()`) и вызывает Кроулера
   (`crawler.py/crawl_single_source(source, keywords)`.)
2. Кроулер отвечает только за получение ссылок на статьи. Посему, вызывает `Strategy.run(source, headers)`. Она в свою
   очередь запускает стратегию из папки `strategies` для конкретного источника:
    - если источник RSS, то вызывает `rss_strategy`:
        - `rss_strategy` с помощью requests получает RSS-ленту, парсит её с помощью `feedparser` и ищет `links`. Собрав
          в список все ссылки, она возвращает его выше.
    - если источник обычный сайт, то вызывает `website_strategy`:
        - `website_strategy` с помощью `requests` получает HTML-код страницы, парсит его с помощью `BeautifulSoup`, ищет
          все ссылки и возвращает их в виде списка.
    - Получив список ссылок `crawl_single_source` инициализирует `pipeline` вместе с сервисами и передает
      список в него для дальнейшей обработки.
3. `pipeline` передает список ссылок в `download_service`:
    - `download_service` с помощью asyncio и aiohttp скачивает по странице по ссылкам, и возвращает список HTML-код
      страниц в виде словарей. После чего `Pipeline` передает список страниц в `extract_service`:
    - `extract_service` в цикле с помощью `trafilatura` извлекает метаданные из страниц, и возвращает список
      `ParsedArticle` с полями (`url`, `text`,  `title`, `published_at`).
      После чего `Pipeline` передает список `ParsedArticle` в `match_service`:
    - `match_service` инициализирует cоздает регулярное выражение для поиска ключевых слов, и в цикле проверяет каждую
      статью (титульник и текст) на наличие ключевых слов. В случае нахождения таковых - добавляет в статью (
      `ParsedArticle`) ключевые слова (`matched_keywords`), и возвращает список статей с найденными ключевыми словами.
      После чего `Pipeline` передает список статей с найденными ключевыми словами в `StorageService`:
    - `storage_service` c `transaction.atomic()` проходится циклом по статьям и если статья уже существует в базе
      данных, она обновляется, иначе создается новая запись (`update_or_create()`). Возвращает количество
      сохраненных статей.
    - После завершения работы `Pipeline` формирует краткую статистику (`PipelineResult`): сколько страниц скачано,
      сколько статей извлечено, сколько найдено совпадений и сколько статей сохранено.

И вот, по завершению работы в лог лисичке пишется "Краулер завершил свою работу, с <...> результатами", ей остается лишь
зайти в своей Админ-Панеле в раздел Статей (`Articles`) и увидеть все найденные новости, которые она может просмотреть,
отредактировать и экспортировать, по желанию.
