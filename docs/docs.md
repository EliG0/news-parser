# Краткое обьяснение работы программы

Представим, что вы лисичка, которая хочет собрать все вкусные рецепты из интернета, но связанных с молоком. Для этого у
вас есть специальная программа, которая умеет искать и собирать определенные рецепты с разных сайтов. Лисичка запускает
программу, по туториалу, и попадает на главный экран. Но интерфейс пока не предусматривает адекватную страницу с UX,
посему лисичка переходит в Админ-Панель. В Админ панеле она заходит в раздел источников (`Sources`), и добавляет туда
ссылки на сайты, опять же по туториалу, с которых она хочет собирать рецепты. Так у лисички получился список источников
с самыми разными типами, от RSS-ленты (например Рецепт48) до обычных сайтов, к которым нужно применять регулярные
выражения (например РецептнаяОбласть). После того как лисичка добавила все источники, она может вернуться на главный
экран и нажать на заветную кнопку.

1. При нажатии на кнопку, в файле `tasks.py` происходит вызов функции
   `celery_crawl_all_active_sources_task()`. Эта функция собирает все активные источники из базы данных в список, и в
   цикле вызывает функцию `celery_crawl_single_source(source_id)`. Она же, в свою очередь, лезет в БД, достает эту
   статью по `source_id`, ищет в кэше кейворды и вызывает функцию `crawl_single_source(source, keywords)` из файла
   `parser/Crawler.py`.
2. `crawl_single_source` вызывает `Strategy.run(source, headers)`. Она в свою очередь запускает стратегию из папки
   `strategies` для конкретного источника:
    - если источник RSS, то вызывает `RSSStrategy`:
        - `RSSStrategy` с помощью requests получает RSS-ленту, парсит её с помощью `feedparser` и ищет `links`. Собрав в
          список все ссылки, она возвращает его выше.
    - если источник обычный сайт, то вызывает `WebsiteStrategy`:
        - `WebsiteStrategy` с помощью requests получает HTML-код страницы, парсит его с помощью `BeautifulSoup`, ищет
          все ссылки и возвращает их в виде списка.
          Получив список ссылок `crawl_single_source` передает инициализирует `pipeline` вместе с сервисами и передает
          список в него для дальнейшей обработки.
3. `Pipeline` передает список ссылок в `DownloadService`:
    - `DownloadService` с помощью asyncio и aiohttp скачивает по странице по ссылкам, и возвращает список страниц в виде
      словарей.
      После чего `Pipeline` передает список страниц в `ExtractService`:
    - `ExtractService` в цикле с помощью `trafilatura` извлекает тело из страниц, и возвращает список `ParsedArticle` с
      полями (`url`, `text`,  `title`).
      После чего `Pipeline` передает список `ParsedArticle` в `MatchService`:
    - `MatchService` инициализирует паттерн для поиска ключевых слов, и в цикле проверяет каждую статью (титульник и
      текст) на наличие ключевых слов. В случае нахождения таковых - добавляет в статью (`ParsedArticle`) ключевые
      слова (`matched_keywords`), и возвращает список статей с найденными ключевыми словами.
      После чего `Pipeline` передает список статей с найденными ключевыми словами в `StorageService`:
    - `StorageService` c `transaction.atomic()` проходится циклом по статьям и с помощью `get_or_create()` сохраняет их
      в БД. Возвращает количество сохраненных статей.
      После завершения всего пайплайна, возвращается `PipelineResult` с полями количества скачанных, извлеченных,
      сматченных и сохраненных статей.

В лог лисичке пишется "Краулер завершил свою работу, с <...> результатами", ей остается лишь радостной зайти в своей
Админ-Панеле в раздел Статей (`Articles`) и увидеть все найденные рецепты, которые она может просмотреть,
отредактировать и экспортировать, по желанию.
